# 🧠 P-Reinforce 위키 저장 스킬 (모든 에이전트용)

> **이 파일은 모든 안티그래비티 에이전트 워크스페이스에 배치됩니다.**
> 대화 중 학습한 내용을 중앙 위키(INFJ-s-wiki)에 자동 저장합니다.

---

## 🎯 언제 실행하는가?

**자동 트리거 조건** — 다음 중 하나라도 해당되면 대화 종료 전에 이 스킬을 실행:

1. 사용자가 명시적으로 요청: "위키에 정리해줘", "오늘 배운 거 정리", "학습 내용 정리", "wiki에 저장", "배운 거 저장"
2. 대화에서 **새로운 인사이트, 패턴, 결정, 스킬**이 도출되었을 때 (에이전트 자율 판단)
3. 사용자가 "오늘 끝" 또는 대화를 마무리할 때

**트리거하지 않는 경우:**
- 단순 잡담, 이전 대화 확인, 설정 변경 같은 비학습 대화
- 디버깅만 하다 끝난 경우 (새로운 인사이트 없음)

---

## 📋 실행 절차

### 1단계: 학습 내용 추출 (에이전트가 수행)

현재 대화를 분석하여 아래 항목을 추출한다. **잡담과 디버깅 과정은 제외**, 핵심 인사이트와 패턴만 뽑는다.

| 필드 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `title` | ✅ | 학습 주제 (한국어, 명확한 제목) | `"ACE-Step 음악 생성 BPM 최적화"` |
| `summary` | ✅ | 한 줄 핵심 요약 | `"145-160 BPM이 최적 리텐션 구간"` |
| `agentName` | ✅ | 이 에이전트의 이름 | `"루나"`, `"알파"`, `"레오"` 등 |
| `agentDomain` | ✅ | 에이전트 도메인 | `"YouTube 음악 채널"`, `"주식 투자"` 등 |
| `keyLearnings` | ✅ | 핵심 배운 점 (배열, 최소 1개) | `["BPM 145가 리텐션 최적", "글리치 이펙트는 역효과"]` |
| `decisions` | 선택 | 이 대화에서 결정한 사항들 | `["1일 3영상 스케줄 확정"]` |
| `skills` | 선택 | 새로 익힌 스킬/패턴/워크플로우 | `["FFmpeg 비트싱크 파이프라인"]` |
| `connections` | 선택 | 관련 기존 지식 키워드 | `["리텐션 전략", "Faceless Noir"]` |

### 2단계: 위키에이전트 `00_Raw/`에 마크다운 저장

아래 경로에 마크다운 파일을 직접 생성한다:

```
C:\Users\YS\Desktop\안티그래피티\위키에이전트\00_Raw\{파일명}.md
```

**파일명 규칙:** `YYYY-MM-DD_에이전트명_제목요약.md`
- 예: `2026-04-22_루나_BPM_최적화_전략.md`

**마크다운 포맷:**

```markdown
# {title}

> 📅 {YYYY-MM-DD} {HH:MM} | {agentName} 에이전트 대화에서 자동 추출
> 🤖 에이전트: {agentName} ({agentDomain})
> 🔗 Conversation: {conversationId}

## 💡 핵심 요약
{summary}

## 📖 오늘 배운 것
- {keyLearnings[0]}
- {keyLearnings[1]}
- ...

## ⚖️ 결정한 사항
- {decisions[0]}
- ...

## 🚀 새로운 스킬/패턴
- {skills[0]}
- ...

## 🔗 연관 지식
- [[{connections[0]}]]
- [[{connections[1]}]]
- ...

---
*Source: {agentName} Agent Conversation Export*
```

### 3단계: `npm run process` 실행

```bash
cd "C:\Users\YS\Desktop\안티그래피티\위키에이전트"
npm run process
```

이 명령이 자동으로:
1. RL 기반 분류 (Projects / Topics / Decisions / Skills 중 배치)
2. Karpathy 위키 템플릿으로 변환
3. 지식 그래프에 노드/엣지 추가 (최소 2개 연결)
4. Git 커밋 + GitHub Push

### 4단계: 결과 검증 및 보고

프로세스 완료 후 사용자에게 보고:
- ✅ 어떤 카테고리에 배치되었는지
- 🔗 몇 개의 기존 지식과 연결되었는지
- 🏆 RL 보상 점수
- 📦 Git 커밋 해시

> ⚠️ **절대 룰**: `npm run process` 실행 후 출력만 믿지 말고, 실제 파일이 `10_Wiki/` 하위에 생성되었는지 독립 검증할 것.

## 🔑 핵심 규칙

### 절대 금지 사항
- ❌ `10_Wiki/`에 직접 파일 추가 금지 (반드시 `00_Raw/` 경유)
- ❌ `git init` 금지 (이미 초기화됨)
- ❌ `git push --force` 금지
- ❌ Notion 검색/저장 금지 (위키에이전트만 사용)

### 반드시 준수
- ✅ 모든 학습 내용은 `00_Raw/`에 `.md` 파일로 저장
- ✅ `npm run process` 또는 `node src/process.js`로 처리
- ✅ 외부 작업(git push 등) 후 반드시 독립 검증
- ✅ 파일 인코딩은 UTF-8

## 💡 품질 가이드라인

### 좋은 학습 기록의 조건
- **구체적 수치/데이터 포함**: "BPM 145가 좋다" → "BPM 145-160 구간에서 평균 리텐션 42%→58% 상승"
- **반복 가능한 패턴**: "이렇게 하면 잘 된다" → "조건 A일 때 전략 B를 적용하면 결과 C가 나온다"
- **실패에서 배운 교훈**: 무엇을 시도했고 왜 실패했는지 기록
- **의사결정의 근거**: 왜 A 대신 B를 선택했는지

### 연결(connections) 키워드 선정 팁
- 이미 위키에 있을 법한 주제와 연결 (예: "리텐션 전략", "SEO 최적화")
- 다른 에이전트의 도메인과 교차하는 개념 (예: 루나의 "음악 BPM" ↔ 레오의 "알고리즘 리텐션")
- 상위 개념으로 연결 (예: "FFmpeg 명령어" → "영상 처리 파이프라인")
