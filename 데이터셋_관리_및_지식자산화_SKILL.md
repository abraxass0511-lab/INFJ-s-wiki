## 📖 PART 1: 데이터 축적 시스템 (챕터 8)

### 1.1 데이터 저장 도구 (하나만 선택)

| 도구 | 장점 | 추천 상황 |
| --- | --- | --- |
| **Google Sheets** ⭐ | 무료, 어디서나 접근 | 가장 추천 |
| Notion DB | 정리·태그 편함 | 글 많이 쓰는 사람 |
| JSON 파일 | 개발자 친화적 | 코딩 가능 |

### 1.2 추천 컬럼 구조

```
| 날짜 | 카테고리 | 질문 | 답변 | 출처 | 상태 |
```

### 1.3 데이터 수집 소스 3가지

**A) 본인 글에서 추출**
- 카톡/슬랙 답장 → Q&A 쌍
- 블로그·SNS 글 → 주제별 질문
- 강의 자료 → 자주 받는 질문
- 메모/일기 → 시그니처 표현

**B) AI 자동 증강 (1개 → 6개)**

프롬프트:
```
이 Q&A를 5가지 다른 질문 표현에 매칭되게 변형해줘.
- 짧은 버전
- 정중한 버전
- 반말 버전
- 상황 추가
- 헷갈리는 사람용

[원본 Q&A]
```

**C) 실사용 피드백 (가장 강력)**

LM Studio로 본인 AI랑 대화하면서:
- 🔴 헛소리한 질문 → 정답 작성해서 추가
- 🔴 톤이 어색한 답변 → 본인 톤으로 다시 써서 추가
- 🔴 답 못한 질문 → 답 만들어서 추가

### 1.4 주기적 발전 루프

```
주 1회 30분 투자:
  월: 이번 주 받은 질문 모으기 (10-15개)
  화: AI 증강으로 30-50개로 부풀리기
  수: 데이터셋에 추가 + 카테고리 균형 점검
  목: 부족한 부분 메모
  금: LM Studio에서 실사용
─────────────────────────────────────
2주마다: 재학습 → 새 GGUF 배포
```

### 1.5 누적 로드맵

| 기간 | 누적 데이터 | 모델 수준 |
| --- | --- | --- |
| 오늘 | 30개 | 기본 페르소나 ✅ |
| 1개월 | 100개 | 자연스러운 응답 |
| 6개월 | 700개 | "이거 본인이네" |

## 📖 PART 2: 지식 자산화 — 노하우와 코드의 데이터화 (챕터 9)

### 2.1 노하우를 데이터화하는 5가지 각도

> 💡 **하나의 노하우 = 10~20개의 Q&A**

| 각도 | 의도 | 예시 질문 |
| --- | --- | --- |
| **직접** (What/How) | 정체성 박기 | "외주 단가 얼마부터?" |
| **상황** (When/Where) | 실전 응용 | "클라가 단가 깎아달래" |
| **초보** (Why) | 친절한 톤 | "처음 어떻게 정해?" |
| **반대** (But) | 신념 강화 | "5만 원은 너무 비싸지 않아?" |
| **잡담** (Feel) | 페르소나 톤 | "단가 정하기 너무 어려워" |

### 2.2 코드/프로젝트를 지식화하기

> ⚠️ 코드 자체는 "지식"이 아니다. 지식 = 의사결정.

**5가지 카테고리로 추출:**

| 카테고리 | 내용 |
| --- | --- |
| **의사결정** (Why) | 왜 이 기술을 골랐는지 |
| **삽질/실수** (Mistakes) | 어디서 막혔는지 |
| **패턴** (How) | 반복 가능한 구조 |
| **함정** (Avoid) | 다른 사람이 실수할 부분 |
| **메타 교훈** (Wisdom) | 코드 너머의 원칙 |

### 2.3 회고 템플릿 (프로젝트당 30분)

```markdown
1) 왜 만들었나? (1줄)
2) 어떤 기술 골랐고 왜?
3) 막혔던 순간 Top 3
4) 다음에 만들면 다르게 할 것 3가지
5) 처음부터 알았으면 좋았을 것 5가지
6) 다른 프로젝트에도 쓸 패턴 3가지
```

> 🎯 이 6개 질문에 답하면 → **30~50개의 Q&A 데이터** 자동 생성

### 2.4 AI 자동 추출 프롬프트

```
이 코드는 내가 짠 것이다. 이 코드를 짤 때 내가 내린 의사결정과
삽질했을 만한 부분을 분석해서, 5가지 카테고리로 골고루 20개 Q&A 생성:

1. 의사결정 (Why)
2. 삽질 (Mistakes)
3. 패턴 (How)
4. 함정 (Avoid)
5. 메타 교훈 (Wisdom)

답변 톤:
- 친근하고 명확한 1인칭
- "당장 해볼 것:" 같은 액션 포함

[코드 붙여넣기]
```

> 🚀 30분 회고가 1분으로 단축

## 🔑 실행 규칙 (에이전트 필수 준수)

### "문서화해줘" 요청 시 처리 순서:

1. **대상 파악** — 문서화할 내용이 "노하우"인지 "코드/프로젝트"인지 판단
2. **각도 전개** — 노하우: 5가지 각도 (직접/상황/초보/반대/잡담) / 코드: 5가지 카테고리 (Why/Mistakes/How/Avoid/Wisdom)
3. **Q&A 쌍 생성** — 최소 10개, 가능하면 20개 이상
4. **AI 증강** — 원본 Q&A 1개당 5가지 변형 생성 (짧은/정중한/반말/상황추가/헷갈리는사람용)
5. **컬럼 형식 저장** — `| 날짜 | 카테고리 | 질문 | 답변 | 출처 | 상태 |`
6. **위키 저장** — WIKI_SAVE_SKILL 절차에 따라 `00_Raw/`에 저장 후 구조화

### 절대 금지
- ❌ 코드를 그대로 복붙해서 "문서화 완료"라고 하지 말 것
- ❌ 의사결정 맥락 없이 결과물만 나열하지 말 것
- ❌ Q&A 없이 단순 요약만 하지 말 것

### 반드시 준수
- ✅ 모든 노하우는 최소 5가지 각도에서 질문을 펼칠 것
- ✅ 코드 지식화 시 반드시 5가지 카테고리 (Why/Mistakes/How/Avoid/Wisdom)를 포함할 것
- ✅ 회고 템플릿 6개 질문을 기반으로 구조화할 것
- ✅ 생성된 Q&A는 추후 파인튜닝 데이터셋으로 활용 가능한 형식으로 정리할 것
