## 경험 구조화 프레임워크 시각화 디자인 가이드라인

이 가이드라인은 '경험 구조화 마스터 클래스'의 핵심인 **'경험 구조화 프레임워크 (Context $\rightarrow$ Analysis $\rightarrow$ Action $\rightarrow$ Result)'**를 시청자에게 명확하고 직관적으로 전달하기 위한 인포그래픽 디자인 방향을 제시합니다.

### 1. 비주얼 시스템 정의 (Visual System Definition)

| 요소 | 설정 값 | 설명 및 의도 |
| :--- | :--- | :--- |
| **브랜드 컬러** | Primary: `#4A5568` (차분한 슬레이트 그레이) | 내향적이고 깊이 있는 사유를 상징하며 신뢰감을 부여합니다. |
| | Secondary: `#A0AEC0` (옅은 회색/블루) | 정보의 흐름과 분석 단계를 시각적으로 연결하는 데 사용됩니다. |
| | Accent: `#718096` (차분한 그레이 블루) | 행동(Action) 단계와 최종 결과(Result)를 강조하여 실행력을 부여합니다. |
| **타이포그래피** | 제목/헤드라인: Pretendard Bold | 명확성과 구조성을 강조합니다. |
| | 본문/설명: Pretendard Regular | 가독성을 최우선으로 합니다. |
| **레이아웃 스타일** | 흐름형 (Flow-based) & 모듈형 (Modular) | 각 단계가 유기적으로 연결되어 있음을 보여주며, 정보의 계층 구조를 명확히 합니다. |
| **이미지 스타일** | 미니멀리즘 다이어그램 (Minimalist Diagram) | 복잡한 경험을 단순화하고, AI 시대에 필요한 '구조화'의 본질을 강조합니다. |

### 2. 인포그래픽 핵심 구조 및 컷별 비주얼 레퍼런스

**전체 컨셉:** **"혼돈(Chaos)에서 질서(Order)로의 전환 (The Transition from Chaos to Order)"**

#### 🖼️ 컷 1: 도입부 - 혼돈의 상태 (Context)
*   **목표:** 시청자의 현재 상태(흩어진 경험)에 공감하며 문제 제기.
*   **비주얼 레퍼런스:**
    *   **배경:** 어둡고 차분한 슬레이트 그레이 배경.
    *   **중앙 요소:** 수많은 작은 메모 조각들, 흩어진 노트, 미완성된 아이디어들이 무질서하게 겹쳐진 이미지 (디지털화된 '혼돈').
    *   **텍스트 오버레이:** "당신의 경험은 어디에 있나요?" 또는 "흩어진 기억의 무게"와 같은 질문을 중앙에 배치.
    *   **색상 사용:** 혼돈 영역은 채도가 낮은 회색 톤으로 표현하여 무거움을 전달.

#### 🖼️ 컷 2: 프레임워크 소개 - 구조화의 시작 (The Framework Introduction)
*   **목표:** 해결책(프레임워크)을 제시하고, 시스템의 존재를 알림.
*   **비주얼 레퍼런스:**
    *   **중앙 요소:** 네 개의 단계($\rightarrow$)가 명확하게 연결된 깔끔한 다이어그램. (Context $\rightarrow$ Analysis $\rightarrow$ Action $\rightarrow$ Result)
    *   **디자인 강조:** 각 박스는 옅은 회색 배경에 Primary 컬러(`\#4A5568`)로 테두리를 처리하여 구조적 느낌을 극대화합니다. 화살표는 Secondary 컬러(`\#A0AEC0`)를 사용하여 흐름을 부드럽게 연결합니다.
    *   **핵심 강조:** '경험 구조화 템플릿 A'라는 키워드를 중앙에 배치하고, 이 프레임워크가 그 중심임을 명시.

#### 🖼️ 컷 3: 단계별 상세 설명 (Context $\rightarrow$ Analysis)
*   **목표:** 첫 번째 단계의 구체적인 방법론 제시.
*   **비주얼 레퍼런스:**
    *   **레이아웃:** 좌측에 'Context' 박스를 배치하고, 우측에 화살표를 통해 'Analysis' 박스로 연결합니다.
    *   **Context (입력):** 메모 조각들이 깔끔하게 정리되어 하나의 큰 블록으로 모이는 모습. (데이터 수집의 시각화)
    *   **Analysis (분석):** 흩어진 데이터가 AI 엔진(추상적인 아이콘)을 통과하며 패턴과 구조로 변환되는 과정을 표현. (AI의 역할 강조)
    *   **색상 사용:** Context는 차분한 회색, Analysis 과정은 Secondary 컬러를 사용하여 정보 처리의 흐름을 시각화합니다.

#### 🖼️ 컷 4: 단계별 상세 설명 (Action $\rightarrow$ Result)
*   **목표:** 실행과 최종 결과의 명확한 대비 제시.
*   **비주얼 레퍼런스:**
    *   **레이아웃:** 'Action' 박스와 'Result' 박스를 수평 또는 수직으로 분리하여 배치합니다.
    *   **Action (실행):** 분석 결과를 바탕으로 구체적인 템플릿(A)을 채워 넣는 행위의 시각화. 손이 직접 구조를 완성하는 듯한 모듈형 디자인. Accent 컬러(`\#718096`)를 사용하여 실행의 의지를 강조합니다.
    *   **Result (결과):** 최종적으로 잘 정돈되고 빛나는 자산(템플릿)이 명확하게 제시되는 모습. 깔끔하고 안정적인 느낌으로 마무리.

### 3. 디자인 가이드 요약 (Execution Checklist)

1.  **스타일 일관성:** 모든 다이어그램은 미니멀하고 선형적이며, 색상 팔레트(`\#4A5568`, `\#A0AEC0`, `\#718096`)를 엄격하게 준수해야 합니다.
2.  **AI 메타포 활용:** '분석' 단계에서는 추상적인 데이터가 AI 엔진을 거쳐 구조화되는 과정을 시각적으로 표현하여, 이 프레임워크가 **AI 기반의 자동화 시스템**임을 암시합니다.
3.  **흐름 강조:** 모든 컷은 왼쪽에서 오른쪽 또는 위에서 아래로 명확하게 흐르는 화살표를 통해 논리적 순서를 강제해야 합니다.
