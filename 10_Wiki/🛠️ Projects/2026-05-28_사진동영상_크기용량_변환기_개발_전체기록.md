### 세부 내용
- **클라이언트 사이드 처리** — 이미지/동영상 변환을 서버 없이 사용자 브라우저에서 수행 (개인정보 보호 + 비용 0원)
- **Canvas API 이미지 압축** — `toDataURL(format, quality)`의 quality를 이진 탐색하여 목표 용량 달성
- **FFmpeg.wasm Worker CORS** — CDN에서 로드한 FFmpeg의 내부 Worker가 same-origin 정책에 의해 차단됨
- **UMD 로컬 호스팅** — Worker 청크 파일(814.ffmpeg.js)을 로컬에 저장하면 CORS 문제 근본 해결
- **import.meta.url 파싱 에러** — UMD에 포함된 import.meta.url은 일반 script 태그에서 SyntaxError 발생 → location.href로 교체
- **GitHub Pages 무료 배포** — 정적 사이트를 `gh` CLI로 즉시 배포, 모바일/PC 어디서든 접속 가능
- `index.html` — 메인 UI 구조 생성 (업로드 → 설정 → 결과 3단계)
- `style.css` — 디자인 시스템 (CSS 변수 기반)
- `app.js` — Canvas API로 이미지 리사이즈 + quality 이진 탐색으로 목표 용량 달성
- 드래그 앤 드롭 업로드, 미리보기, 원본 vs 결과 비교, 다운로드 기능 구현
- `index.html` — 제목 변경, 비율 잠금 버튼 제거
- `style.css` — 다크 테마 → 화이트 테마로 전면 변경 (CSS 변수 값 수정)
- `app.js` — 가로/세로 연동 로직 제거, 독립 입력으로 변경
- FFmpeg.wasm 라이브러리를 사용하면 브라우저에서 동영상 리사이즈/압축 가능
- 해상도 변경, 용량 압축, 포맷 변환 모두 가능
- 다만 사용자 PC의 CPU를 사용하므로 큰 파일은 시간이 오래 걸림
- 브라우저 메모리 한계로 매우 큰 파일(2GB+)은 어려울 수 있음
- 완전 무료 (오픈소스 라이브러리, 서버 비용 없음)
- 해상도 변경(세로→가로 포함), 용량 압축 모두 가능
- Git 설정 확인: `abraxass0511-lab` / `abraxass0511@gmail.com`
- 구현 계획(implementation_plan.md) 작성:
- - 탭 UI 추가 (사진/동영상)
- - FFmpeg.wasm으로 동영상 변환
- - coi-serviceworker.js로 SharedArrayBuffer 지원
- - GitHub Pages 배포
- - 최종 URL: `https://abraxass0511-lab.github.io/image-video-resizer`
- "사진 / 동영상" 탭 네비게이션 추가
- 기존 이미지 섹션을 사진 탭(#imageTab) 래핑
- 동영상 탭(#videoTab) 추가: 업로드 → 설정(해상도 프리셋 1080p~240p, 용량 1MB~100MB) → 프로그레스바 → 결과 비교 → 다운로드
- FFmpeg.wasm 초기화/로드
- 비트레이트 자동 계산: `bitrate = (target_bytes × 8) / duration - audio_bitrate`
- 짝수 해상도 보정 (H.264 요구사항)
- FFmpeg exec: scale + bitrate + libx264 + aac + faststart
- **"FFmpeg.wasm은 CDN에서 바로 쓸 수 있다" 오해:** ESM dynamic import → Worker CORS 차단 → UMD 로컬 호스팅 필수
- **"classWorkerURL로 해결 가능" 부분적 오류:** ESM worker의 내부 import가 blob URL에서 작동 안 함
- **"import.meta.url은 UMD에 없다" 실수:** 원본 UMD 빌드에 포함되어 있어 일반 script에서 SyntaxError
- **"toBlobURL이 만능" 오해:** Worker 내부의 추가 import까지는 해결 못함
- **coi-serviceworker vs CORS 혼동:** coi-serviceworker = COEP/COOP(SharedArrayBuffer), CORS Worker = same-origin 정책 — 별개 문제
- **4번 실패 교훈:** "로컬 파일 호스팅 + UMD 빌드 + import.meta 제거" 3가지 조건 모두 충족 필요
