### 세부 내용
- **단일 도구 원칙** — 저장 도구는 하나만 골라서 꾸준히 쓴다 (Google Sheets 추천)
- **컬럼 구조 표준화** — `| 날짜 | 카테고리 | 질문 | 답변 | 출처 | 상태 |`
- **3소스 수집** — 본인 글 추출 + AI 자동 증강 + 실사용 피드백
- **AI 증강 공식** — 1개 Q&A → 5가지 변형(짧은/정중한/반말/상황추가/헷갈리는사람용) → 6개
- **실사용 피드백 루프** — LM Studio로 대화하면서 오답·어색한톤·미답변을 직접 수정 추가
- **주간 30분 루프** — 월(수집) → 화(증강) → 수(추가) → 목(점검) → 금(실사용)
- **2주 재학습 사이클** — 재학습 → 새 GGUF 배포
- **HF 백업 패턴** — `dataset.push_to_hub(private=True)` → 새 모델 출시 시 1줄 재학습
- **데이터 저장 도구 비교:**
- **데이터 수집 소스 A — 본인 글에서 추출:** 카톡/슬랙 답장 → Q&A 쌍, 블로그·SNS → 주제별 질문, 강의 자료 → 자주 받는 질문, 메모/일기 → 시그니처 표현
- **데이터 수집 소스 B — AI 자동 증강:** ChatGPT/Claude에 "이 Q&A를 5가지 다른 질문 표현에 매칭되게 변형해줘"로 요청 → 1개 → 6개로 부풀리기
- **데이터 수집 소스 C — 실사용 피드백 (가장 강력):** LM Studio에서 헛소리한 질문 → 정답 작성, 톤 어색 → 본인 톤 재작성, 답 못한 질문 → 답 만들어서 추가
- **누적 로드맵:**
- **HuggingFace 백업 코드:**
- **"모델이 핵심이다" 오해:** 모델은 소비재(6개월마다 교체), 데이터가 진짜 자산 → 데이터 축적 시스템이 모델 선택보다 우선
- **"데이터가 많으면 좋다" 오해:** 나쁜 데이터 1000개 < 좋은 데이터 30개 → 품질 우선, AI 증강으로 양 보완
