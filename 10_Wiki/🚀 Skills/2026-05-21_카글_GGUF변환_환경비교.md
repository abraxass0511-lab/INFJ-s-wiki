## 1️⃣ 핵심 개념 이해

### 모델 형식 비교 (LoRA vs 병합 vs GGUF)

| 형식 | 추가 학습 | 추론(대화) | 다른 모델 적용 | 크기 | 비유 |
|---|---|---|---|---|---|
| **LoRA** (.safetensors) | ✅ 가능 | ⚠️ 원본 필요 | ✅ 가능 | ~200MB | 스티커 (떼서 다른 노트에 붙임) |
| **병합 16bit** | ✅ 가능 | ✅ 가능 | ❌ 분리 불가 | ~5GB | 코팅한 완성품 |
| **GGUF** | ❌ 불가능 | ✅ LM Studio | ❌ 불가 | ~1.5GB | 실행 파일(.exe) |

### LoRA를 별도로 보관하는 이유

| 이유 | 설명 |
|---|---|
| **빠른 백업** | 200MB = 10초 업로드 (세션 끊기기 전에 저장 가능!) |
| **용량 절약** | 200MB vs 5GB (25배 차이) |
| **유연성** | 다른 베이스 모델에도 적용 가능 (Gemma → Llama 등) |
| **버전 관리** | v1, v2, v3... 이어서 학습 가능 |

### 양자화 옵션

| 양자화 | 크기 | 품질 | 비유 |
|---|---|---|---|
| `f16` (16bit) | ~5 GB | ⭐⭐⭐⭐⭐ 최고 | 원본 사진 |
| `q8_0` (8bit) | ~2.5 GB | ⭐⭐⭐⭐ 좋음 | 고화질 JPEG |
| **`q4_k_m` (4bit)** | **~1.5 GB** | **⭐⭐⭐ 적당** | **일반 JPEG ← 추천** |
| `q2_k` (2bit) | ~0.8 GB | ⭐⭐ 낮음 | 저화질 JPEG |

### .ipynb vs .py 파일

| | .py | .ipynb |
|---|---|---|
| 내용 | 코드만 | 코드 + 출력 결과 + 메모 |
| 실행 | 터미널 `python 파일.py` | 코랩/카글에서 셀 단위 |
| 용도 | 자동화, 배포 | 실험, 학습, 공유 |
| 셀이 많을 때 | 하나의 파일 (순서대로 실행) | 셀 단위로 분리 실행 |

## 4️⃣ 학습 결과 & 파라미터

### 학습 파라미터 (어제 동일 유지)

| 파라미터 | 값 | 이유 |
|---|---|---|
| `r` (LoRA rank) | 16 | rank 높을수록 표현력 ↑ |
| `lora_alpha` | 16 | rank와 동일하게 |
| `max_steps` | 1000 | 45개 데이터 × ~90회 |
| `learning_rate` | 1e-3 | LoRA 최적값 |
| `gradient_accumulation_steps` | 4 | 실질 배치 4 |

### 학습 결과 비교 (코랩 vs 카글)

| 항목 | 어제 (코랩) | 오늘 (카글) |
|---|---|---|
| 환경 | Colab T4 x1 | Kaggle T4 x2 |
| loss | 0.2473 | 0.2543 |
| 학습 시간 | 28.8분 | 30.6분 |
| 페르소나 내재화 | ✅ 성공 | ✅ 성공 |
| 키워드 반응 | ✅ | ✅ |
| 대화 자연스러움 | ⚠️ 반복 심함 | ✅ 파라미터 튜닝으로 해결 |

### 추론(대화) 파라미터 최적화

학습 완료 후 대화 품질 문제 발생 → 파라미터 튜닝으로 해결:

```
문제: "1인 기업가... 1인 기업가... 1인 기업가..." 반복 현상
원인: 학습 1000 step → 키워드를 너무 강하게 학습 → 반복 출력
```

#### 해결 과정 (5단계 시행착오)

```
시도 1: repetition_penalty=1.3 (기본)       → 여전히 반복 심함
시도 2: do_sample=True + temp=0.7 + top_p=0.9 → "안녕" 질문에 같은 문장 2번 반복
시도 3: repetition_penalty=1.5 (강화)       → 개선, 일부 반복 남음
시도 4: no_repeat_ngram_size=3 추가         → 너무 강해서 자연스러운 표현도 차단
시도 5: no_repeat_ngram_size=6 (완화)       → ✅ 최적! 자연스러운 대화 + 반복 방지
```

#### 추론 파라미터 의미

| 파라미터 | 값 | 의미 |
|---|---|---|
| `do_sample=False` (Greedy) | - | 항상 확률 1위 단어만 선택 → 반복 발생 |
| `do_sample=True` (Sampling) | - | 확률에 따라 랜덤 선택 → 다양한 출력 |
| `temperature` | 0.7 | 낮을수록 보수적, 높을수록 창의적 |
| `top_p` | 0.9 | 상위 90% 확률 토큰 중에서만 선택 |
| `repetition_penalty` | 1.5 | 이미 출력된 토큰의 확률을 1.5배 감소 |
| `no_repeat_ngram_size` | 6 | 6단어 이상 연속 같은 구절 반복 차단 |

#### 최종 chat 함수 코드

```python
def chat(prompt, max_tokens=200):
    msg = [{"role": "user", "content": [{"type": "text", "text": prompt}]}]
    inp = tokenizer.apply_chat_template(
        msg, add_generation_prompt=True, return_tensors="pt",
    ).to("cuda")
    out = model.generate(
        **inp, max_new_tokens=max_tokens,
        do_sample=True,
        temperature=0.7,
        top_p=0.9,
        pad_token_id=tokenizer.eos_token_id,
        repetition_penalty=1.5,
        no_repeat_ngram_size=6,
    )
    return tokenizer.decode(out[0], skip_special_tokens=True).split("model\n")[-1]
```

### GGUF 변환 실패 → 해결 과정

```

```

### GGUF 변환 최종 코드 (카글 4셀 구성)

```python
# 셀 1: 패키지 설치
!pip install unsloth
!pip install --no-deps transformers==5.5.0 "tokenizers>=0.22.0,<=0.23.0"
!pip install torchcodec
!pip install --no-deps --upgrade timm

import torch

```

```python
# 셀 2: 모델 로드
from kaggle_secrets import UserSecretsClient
from huggingface_hub import login
login(token=UserSecretsClient().get_secret("HF_TOKEN"))

from unsloth import FastModel
model, tokenizer = FastModel.from_pretrained(
    model_name = "INFJ-FATHER/ai-mentor-jay",
    dtype = None,
    max_seq_length = 1024,
    load_in_4bit = True,
)
print("✅ 모델 로딩 완료!")
```

```python
# 셀 3: GGUF 변환 (/tmp 사용으로 디스크 부족 해결)
import os
os.chdir("/tmp")

model.save_pretrained_gguf(
    "/tmp/ai-mentor-jay-gguf",
    tokenizer,
    quantization_method="q4_k_m",
    temporary_location="/tmp",
)
print("✅ GGUF 변환 완료!")
```

```python
# 셀 4: GGUF 파일 찾기 + 허깅페이스 업로드
import glob
gguf_files = glob.glob("/tmp/**/*.gguf", recursive=True)
print("GGUF 파일 위치:", gguf_files)

from huggingface_hub import HfApi
api = HfApi()
api.create_repo("INFJ-FATHER/ai-mentor-jay-GGUF", private=True, exist_ok=True)

# ⚠️ GGUF 파일은 _gguf 접미사 폴더에 저장됨!
api.upload_folder(
    folder_path="/tmp/ai-mentor-jay-gguf_gguf",  # ← _gguf 접미사!
    repo_id="INFJ-FATHER/ai-mentor-jay-GGUF",
)
print("✅ 허깅페이스 업로드 완료!")
```

### GGUF 변환 대안 비교

| 방법 | 난이도 | 안정성 | 속도 | 비용 |

## 6️⃣ LM Studio 로컬 배포

### GGUF 파일 구성 (Gemma 4 멀티모달)

| 파일 | 크기 | 용도 |
|---|---|---|
| `gemma-4-e2b-it.Q4_K_M.gguf` | 3.43 GB | 메인 모델 (텍스트 대화) ⭐ |
| `gemma-4-e2b-it.F16-mmproj.gguf` | 986 MB | 비전 프로젝터 (이미지 분석) |

#### mmproj(멀티모달 프로젝터)란

```
Gemma 4 = 텍스트 두뇌(Q4_K_M) + 눈(mmproj)

Q4_K_M.gguf     → 텍스트 대화 담당 🧠
F16-mmproj.gguf → 이미지 인식 담당 👁️

이미지 분석 능력 → Gemma 4 원본에서 온 것 (구글이 학습)
멘토 페르소나   → 우리가 학습시킨 것 (LoRA)
```

### 모델 설치 경로

```
C:\Users\YS\.lmstudio\models\
├── INFJ-FATHER\                    ← 허깅페이스 유저명
│   └── ai-mentor-jay-GGUF\        ← 모델명
│       ├── gemma-4-e2b-it.Q4_K_M.gguf       ← 메인 (필수)
│       └── gemma-4-e2b-it.F16-mmproj.gguf   ← 비전 (같은 폴더면 자동 인식)
```

### LM Studio 사용법

- 폴더 구조: `유저명/모델명/파일.gguf` (허깅페이스와 동일)
- **자동 인식**: models 폴더에 .gguf 파일 넣으면 자동으로 목록에 표시
- **mmproj 자동 연결**: Q4_K_M만 로드해도 같은 폴더의 mmproj 자동 인식 → 이미지 분석 가능
- **mmproj는 따로 로드 불가**: 보조 파일(플러그인)이라 단독 실행 안 됨

### 카글 → .py 변환 방법

```
카글 노트북 → "Link to Colab" 클릭 → 코랩에서 열림 → 파일 → .py 다운로드
```
