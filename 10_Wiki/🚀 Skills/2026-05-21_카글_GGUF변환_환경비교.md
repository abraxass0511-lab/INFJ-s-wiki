### 세부 내용
- **환경 전환** — 코랩 RAM(12.7GB) 부족 시 카글(30GB)로 이동하면 해결
- **디스크 우회** — 카글 working(20GB) 부족 시 `/tmp`(수백 GB) 폴더 활용
- **역할 분담** — 학습(GPU 필요) = 카글, GGUF 변환(안정성) = 허깅페이스
- **이중 업로드** — LoRA(패치) + 병합 16bit(완성품) 둘 다 허깅페이스 보관
- **자동 인식** — LM Studio는 같은 폴더의 mmproj를 자동 연결 (이미지 분석 가능)
- **추론 파라미터 최적화** — `do_sample=True` + `repetition_penalty=1.5` + `no_repeat_ngram_size=6` 조합이 최적
- 폴더 구조: `유저명/모델명/파일.gguf` (허깅페이스와 동일)
- **자동 인식**: models 폴더에 .gguf 파일 넣으면 자동으로 목록에 표시
- **mmproj 자동 연결**: Q4_K_M만 로드해도 같은 폴더의 mmproj 자동 인식 → 이미지 분석 가능
- **mmproj는 따로 로드 불가**: 보조 파일(플러그인)이라 단독 실행 안 됨
- **코랩 한계 확인:** RAM 12.7GB로 GGUF 변환(~14GB 필요) 시 OOM 발생 → 카글로 해결
- **카글 디스크 한계:** working 폴더 20GB 부족 → `/tmp` + `temporary_location` 조합으로 해결
- **GGUF 파일 경로 주의:** `save_pretrained_gguf`의 출력은 `_gguf` 접미사 폴더에 저장됨
- **업로드 실수 → 전화위복:** 잘못된 폴더 업로드로 16bit 병합 모델이 올라감 → GGUF 재변환 가능한 자산 확보
- **최적 전략 발견:** 학습=카글(GPU), 변환=허깅페이스(안정), 실행=LM Studio(로컬)
- **반복 문제 해결:** `do_sample=False`(greedy) → `do_sample=True` + `repetition_penalty=1.5` + `no_repeat_ngram_size=6`
- **chat 함수 3개 주의:** 코드에 chat 함수가 3곳(Line 875, 914, 940)에 있어 혼동 → 실제 실행되는 마지막 정의를 수정해야 함
- **2B 모델 한계 인식:** 26억 파라미터로는 깊이·자연스러움 부족 → 데이터 품질로 보완
- **카글 %%capture 제거 권장:** 코랩과 달리 카글에서는 %%capture 제거하고 로그 직접 확인이 더 안정적
