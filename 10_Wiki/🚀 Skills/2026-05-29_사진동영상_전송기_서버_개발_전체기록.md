### 세부 내용
- **FastAPI 멀티파트 업로드** — `UploadFile`로 500MB까지 사진/동영상 수신, 사진과 동영상 분기 처리 (EXIF/썸네일/얼굴인식은 사진만)
- **SHA-256 해시 중복방지** — 파일 해시로 DB에서 중복 확인, 단 실제 파일이 삭제되었으면 DB 정리 후 재전송 허용
- **외장하드 자동 이관** — psutil로 5초마다 드라이브 마운트 감시, 연결 감지 시 버퍼→외장하드 자동 복사
- **폴더 감시** — 웹 업로드뿐 아니라 폴더에 직접 넣은 파일도 자동 이관 (MEDIA_EXTENSIONS 필터)
- **안드로이드 파일명 문제** — 브라우저 파일 피커가 content provider ID(숫자)를 보냄 → EXIF 날짜 기반 `IMG_YYYYMMDD_HHMMSS` 형식으로 복원
- **Pydantic Optional 필드** — 동영상은 썸네일이 없으므로 `thumbnail_path: str | None = None`으로 선언 필수
- **PowerShell 한글 경로** — cmd.exe batch 파일은 한글 경로 인코딩 깨짐 → .ps1 스크립트 + 바탕화면 바로가기(.lnk)로 우회
- **웹 vs 네이티브 앱 한계** — 웹 브라우저는 보안상 갤러리 접근/파일 삭제/썸네일 생성 불가 → Flutter 앱 필요
- `config.py` — `TEMP_BUFFER_PATH = "C:\Users\YS\Desktop\사진모음집"`, `EXTERNAL_DRIVE_PATH = "X:/"`
- `buffer_service.py` — 파일을 버퍼 폴더에 저장, 외장하드 연결 시 이관 큐 처리
- `drive_watcher.py` — psutil로 5초 간격 외장하드 감지, 연결 시 자동 이관 트리거
- `index.html` — 전면 재작성: 탭 UI 제거, 단일 멀티파일 업로드 화면
- 드래그 앤 드롭 + 파일 목록 + 전송 버튼 + 저장 위치 상태 표시
- 화이트 테마 적용 (보라색 포인트)
- 사진·동영상 함께 선택 가능 (`accept="image/*,video/*"`)
- 웹 업로드 파일 → 날짜_장소명 폴더
- 직접 넣은 파일 → `X:\직접전송\` 폴더
- `169.254.x.x` (APIPA) 제외 필수 → 실제 Wi-Fi IP만 표시
- 핸드폰과 노트북이 같은 Wi-Fi에 연결되어야 접속 가능
- ✅ 노트북 켜짐 + 서버 실행 중 + 같은 Wi-Fi → 전송 가능
- ❌ 노트북 꺼짐 → 서버도 꺼짐 → 접속 불가
- **"웹으로 모든 것이 가능하다" 오해:** 웹 브라우저는 보안 정책으로 갤러리 접근/파일 삭제 불가 → 네이티브 앱 필수
- **"원본 파일명이 유지된다" 오해:** 안드로이드 브라우저는 content provider ID를 보냄 → 서버에서 EXIF 기반 복원 필요
- **"batch 파일이면 충분하다" 오해:** cmd.exe는 한글 경로 인코딩 깨짐 → PowerShell + 바로가기 필요
- **"DB 해시 중복 체크면 충분하다" 오해:** 파일이 삭제되어도 DB 기록이 남아 재전송 차단 → 실제 파일 존재 확인 필수
- **"외장하드 감시만으로 충분하다" 오해:** 웹 업로드 파일만 DB에 등록됨 → 폴더 직접 파일도 감시하는 기능 추가 필요
- **"169.254.x.x도 유효한 IP" 오해:** APIPA(자가 할당 IP)로 외부 기기에서 접근 불가 → Wi-Fi IP만 표시
