### 세부 내용
- **LoRA 파인튜닝** — 전체 모델이 아닌 일부 가중치만 학습 → 메모리 효율적
- **GGUF 변환** — 학습된 모델을 하나의 파일로 압축 → LM Studio에서 로컬 실행
- **토큰 정렬** — `.removeprefix('<bos>')` 제거로 학습/추론 구조 일치 필수
- **키워드 내재화** — 학습 데이터 내 핵심 키워드 노출 빈도 ↑ = 모델 출력 확률 ↑
- **이중 저장 전략** — GGUF(사용/판매) + LoRA(추가 학습) 항상 둘 다 저장
- - 모델: unsloth/gemma-3-4b-it
- - max_steps: 1000 (샘플당 ~90회 학습)
- - learning_rate: 1e-3
- - batch_size: 1, gradient_accumulation: 4
- - GGUF: model.save_pretrained_gguf() → 판매/사용용 (5GB)
- - LoRA: model.save_pretrained() → 추가 학습용 (200MB)
- - 로컬: LM Studio에 GGUF 로드
- - 판매: Gumroad(결제) + HuggingFace(관리/홍보)
- "솔로프리너", "1인 기업가" 등 핵심 키워드를 **45개 중 12개 이상** 답변에 삽입
- 이전 실패: 30개 중 2개 → ~130회 노출 (부족)
- 이번 목표: 45개 중 12개 → ~1,000회 노출 (충분)
- **인터넷 불필요** → 비행기 모드, 산속, 지하철 어디서든 작동
- **데이터 외부 전송 없음** → 개인정보 보안 완벽
- **월 구독료 없음** → 한번 만들면 무료 사용
- **내 코드 보안** → GGUF에는 가중치(숫자)만 포함, 학습 코드/데이터 노출 불가
- **초기 실패 원인:** max_steps=200, 키워드 노출 부족 → 모델이 "Gemma 4" 정체성 유지
- **해결:** max_steps=1000 + 키워드 빈도 7배 증가 → 페르소나 내재화 성공
- **코랩 무료 제한:** 런타임 3~4시간, GPU 메모리 제한 → 학습 직후 즉시 저장 필수
- **GGUF는 추가 학습 불가** → 반드시 LoRA도 함께 저장해야 버전 업데이트 가능
