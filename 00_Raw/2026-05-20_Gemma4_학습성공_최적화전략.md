# [[Gemma 4 파인튜닝 — 학습 성공 및 최적화 전략]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 키워드 밀도 5.7배 강화 + 8가지 하이퍼파라미터 최적화로 페르소나 내재화에 성공했으며, 과적합(반복 출력)은 repetition_penalty=1.3으로 해결한다.

## 📖 구조화된 지식 (Synthesized Content)

### 학습 성공 — 8가지 변경 사항

| # | 항목 | 원본 | 성공 | 효과 |
|---|---|---|---|---|
| ① | `r` / `lora_alpha` | 8 / 8 | **32 / 64** | 학습 용량 4배 ↑ |
| ② | 학습 데이터 | 30개, ~57회 노출 | **45개, ~326회 노출** | 키워드 밀도 5.7배 ↑ |
| ③ | `.removeprefix('<bos>')` | 있음 | **삭제** | 학습/추론 토큰 정렬 |
| ④ | `gradient_accumulation_steps` | 4 | **2** | 업데이트 빈도 2배 ↑ |
| ⑤ | `max_steps` | 30 | **1000** | 학습량 33배 ↑ |
| ⑥ | `learning_rate` | 2e-4 | **1e-3** | 학습 강도 5배 ↑ |
| ⑦ | `lr_scheduler_type` | linear | **cosine** | 수렴 안정성 ↑ |
| ⑧ | `repetition_penalty` | 없음 | **1.3** | 반복 출력 방지 |

### 키워드 밀도 강화 전략

```
핵심 원리: 같은 키워드가 다양한 문맥에서 반복 등장
  → 모델이 "어떤 상황에서든 이 키워드를 출력해야 한다"고 학습
  → 페르소나 내재화
```

| 키워드 | 원본 (30개) | 강화 (45개) | 배수 |
|---|---|---|---|
| AI 멘토 제이 | ~15회 | **75회** | 5배 |
| 1인 기업 | ~20회 | **72회** | 3.6배 |
| 솔로프리너 | ~5회 | **48회** | 9.6배 |
| AI 활용 | ~8회 | **45회** | 5.6배 |
| 생산성 | ~3회 | **36회** | 12배 |
| 당장 해볼 것 | ~3회 | **22회** | 7배 |
| 차근차근 | ~3회 | **28회** | 9배 |

### 데이터 카테고리 설계

| 카테고리 | 개수 | 역할 |
|---|---|---|
| 정체성·인사 | 10개 | "나는 누구인가" — 어떤 질문에든 제이로 답하게 |
| 1인 기업 | 12개 | 도메인 지식 + 키워드 연결 강화 |
| AI 활용 | 10개 | 전문성 + "제이" 연결 강화 |
| 멘탈·태도 | 6개 | 톤/말투 + 키워드 유지 |
| **엣지 케이스** | **7개** | **학습 범위 밖 질문에서 페르소나 이탈 방지** |

### 엣지 케이스가 작동하는 원리

```
LLM은 이미 "이건 전문 분야 밖 질문이다"를 분류하는 능력이 있음
  → 엣지 케이스 7개로 "밖 질문 → AI 멘토 제이 + 전환" 메타 규칙 학습
  → 학습에 없는 질문이 와도 같은 패턴으로 응답 (일반화)

엣지 케이스 0개: "날씨 어때?" → Gemma 정체성 복귀 (페르소나 이탈)
엣지 케이스 7개: "날씨 어때?" → "AI 멘토 제이는 날씨는 모르지만..." (유지!)
```

### 과적합 진단 및 해결

```
Loss 0.3~0.6  → ✅ 적당히 학습 (자연스러운 답변)
Loss 0.2473   → ⚠️ 과적합 시작 (반복 증상 발생)
Loss 0.1 이하  → ❌ 완전 과적합 (앵무새처럼 반복)

증상: "1인 기업 시작은... 1인 기업 시작은... 1인 기업 시작은..."
원인: 모델이 키워드를 "너무 잘" 외워서 같은 표현을 반복 출력
해결: repetition_penalty=1.3 → 이미 출력한 토큰의 확률을 낮춤
```

### LoRA vs GGUF — 저장 전략

| | LoRA | GGUF |
|---|---|---|
| 용도 | 추가 학습, 백업 | LM Studio 사용 |
| 크기 | ~200MB (3개 파일) | ~5GB (1개 파일) |
| 추가 학습 | ✅ 가능 | ❌ 불가능 |
| LM Studio | ❌ 못 씀 | ✅ 바로 실행 |
| 비유 | 게임 세이브 파일 | 완성된 게임 |

### 코랩 RAM 관리 — 크래시 방지

```
❌ 학습 + GGUF 연속 (같은 세션)
  학습 후 RAM: ~9GB + GGUF 변환: +5GB = 14GB > 12.7GB 한계 💥

✅ 학습 → 저장 → 세션 재시작 → GGUF (분리)
  재시작 후 RAM: ~0GB + LoRA 로드: ~4GB + GGUF: +5GB = 9GB < 12.7GB ✅

주의: 세션 크래시(OOM) 시 GPU 한도에 추가 페널티 차감됨
```

### 저장 순서 (안전한 순서)

```
① LoRA 저장 (model.save_pretrained)
② 허깅페이스 업로드 (push_to_hub) ← 안전 확보!
③ 세션 다시 시작 (메모리 초기화)
④ 패키지 재설치 + 허깅페이스에서 LoRA 불러오기
⑤ GGUF 변환 (메모리 여유로운 상태에서)

⚠️ 절대 ②번 전에 ③번 하지 말 것!
```

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **어제 실패 원인:** max_steps=200, 키워드 노출 부족, removeprefix 사용 → 모델이 "Gemma 4" 정체성 유지
- **오늘 해결:** 8가지 파라미터 최적화 + 키워드 밀도 5.7배 증가 → 페르소나 내재화 성공
- **과적합 발견:** loss 0.2473에서 반복 출력 발생 → repetition_penalty=1.3으로 해결
- **코랩 크래시:** 학습 + GGUF를 같은 세션에서 하면 RAM 초과 → 세션 분리 필수
- **저장 순서 교정:** LoRA → Drive 백업 → GGUF (X) → LoRA → 허깅페이스 업로드 → 세션 재시작 → GGUF (O)
- **구글 드라이브 → 허깅페이스:** 무료 용량 제한 없음, 코드 더 간결, 버전 관리 용이
