# [[Gemma 4 파인튜닝 — 로컬 LLM 제작부터 판매까지]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 오픈소스 모델(Gemma 4)을 코랩에서 LoRA 파인튜닝하고, GGUF로 변환하면 인터넷 없이 작동하는 나만의 AI를 만들어 판매할 수 있다.

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴
- **LoRA 파인튜닝** — 전체 모델이 아닌 일부 가중치만 학습 → 메모리 효율적
- **GGUF 변환** — 학습된 모델을 하나의 파일로 압축 → LM Studio에서 로컬 실행
- **토큰 정렬** — `.removeprefix('<bos>')` 제거로 학습/추론 구조 일치 필수
- **키워드 내재화** — 학습 데이터 내 핵심 키워드 노출 빈도 ↑ = 모델 출력 확률 ↑
- **이중 저장 전략** — GGUF(사용/판매) + LoRA(추가 학습) 항상 둘 다 저장

### 학습 파이프라인 (전체 워크플로우)

```
① 데이터 준비 (대화 45개+, 키워드 반복 삽입)
     ↓
② 코랩에서 Unsloth 기반 LoRA 파인튜닝
   - 모델: unsloth/gemma-3-4b-it
   - max_steps: 1000 (샘플당 ~90회 학습)
   - learning_rate: 1e-3
   - batch_size: 1, gradient_accumulation: 4
     ↓
③ 저장 (학습 직후 즉시!)
   - GGUF: model.save_pretrained_gguf() → 판매/사용용 (5GB)
   - LoRA: model.save_pretrained() → 추가 학습용 (200MB)
     ↓
④ 배포
   - 로컬: LM Studio에 GGUF 로드
   - 판매: Gumroad(결제) + HuggingFace(관리/홍보)
```

### 핵심 하이퍼파라미터 설정

| 파라미터 | 값 | 이유 |
|----------|-----|------|
| `max_steps` | 1000 | 45개 데이터 × ~90회 반복 학습 |
| `learning_rate` | 1e-3 | LoRA 가중치를 충분히 변화시킴 |
| `gradient_accumulation` | 4 | 실질 배치 4, 메모리 절약 |
| `warmup_steps` | 5 | 초기 학습 안정성 |

### 토큰 정렬 해결법

```python
# ❌ 잘못된 방식 (토큰 불일치)
tokenizer.apply_chat_template(...).removeprefix('<bos>')

# ✅ 올바른 방식 (토큰 일치)
tokenizer.apply_chat_template(...)  # <bos> 유지
```

### 키워드 내재화 전략

- "솔로프리너", "1인 기업가" 등 핵심 키워드를 **45개 중 12개 이상** 답변에 삽입
- 이전 실패: 30개 중 2개 → ~130회 노출 (부족)
- 이번 목표: 45개 중 12개 → ~1,000회 노출 (충분)

### GGUF vs LoRA 비교

| | GGUF | LoRA |
|---|---|---|
| 용도 | 사용/판매 | 추가 학습 |
| 합쳐진 상태 | ✅ 완성품 | ❌ 베이스 필요 |
| 크기 | ~5GB | ~200MB |
| LM Studio | ✅ 바로 실행 | ❌ 못 씀 |
| 추가 학습 | ❌ 불가 | ✅ 가능 |

### 로컬 LLM 특성

- **인터넷 불필요** → 비행기 모드, 산속, 지하철 어디서든 작동
- **데이터 외부 전송 없음** → 개인정보 보안 완벽
- **월 구독료 없음** → 한번 만들면 무료 사용
- **내 코드 보안** → GGUF에는 가중치(숫자)만 포함, 학습 코드/데이터 노출 불가

### 판매/배포 전략

```
초기: 파일 직접 판매 (비용 0원)
├── Gumroad: 수수료 10%, 자동 파일 전달
├── 크몽: 수수료 20%, 국내 고객
└── HuggingFace: 무료 홍보/데모 + 비공개 관리

후기: API 서비스 전환
├── 서버 필요 (월 3~7만원)
├── 보안 강함 (파일 유출 없음)
└── 인터넷 필수 (고객 ↔ 서버 연결)
```

### 플랫폼 역할 분담

| 플랫폼 | 역할 |
|--------|------|
| **HuggingFace** | AI 전용 GitHub (관리/저장/홍보/버전관리, 무료 무제한) |
| **Gumroad** | AI 전용 쇼핑몰 (결제/자동 전달, 수수료 10%) |
| **Google Drive** | 백업용 (무료 15GB, LoRA 70개 저장 가능) |

### 추가 학습 (이어서 훈련) 워크플로우

```
다음날 코랩에서:
① model = FastModel.from_pretrained("unsloth/gemma-3-4b-it")  ← 베이스 다운
② model.load_adapter("/content/drive/MyDrive/테스트1")         ← LoRA 불러오기
③ 새 데이터 추가 → 학습 → LoRA 다시 저장
④ 최신 LoRA만 유지 (이전 버전 삭제 OK)
```

### 코랩 실행 순서 (내일용)

```
① 설치 셀
② 모델 로딩 셀
③ 데이터 셀 (30개)
④ 추가 데이터 셀 (15개 extend)
⑤ Dataset 변환 셀
⑥ 채팅 템플릿 셀
⑦ SFTTrainer 셀 (max_steps=1000)
⑧ 마스킹 셀
⑨ 학습 셀 (~50분)
⑩ GGUF + LoRA 저장 셀
⑪ chat 테스트
❌ 3️⃣ 셀은 절대 실행 금지!
```

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **초기 실패 원인:** max_steps=200, 키워드 노출 부족 → 모델이 "Gemma 4" 정체성 유지
- **해결:** max_steps=1000 + 키워드 빈도 7배 증가 → 페르소나 내재화 성공
- **코랩 무료 제한:** 런타임 3~4시간, GPU 메모리 제한 → 학습 직후 즉시 저장 필수
- **GGUF는 추가 학습 불가** → 반드시 LoRA도 함께 저장해야 버전 업데이트 가능
