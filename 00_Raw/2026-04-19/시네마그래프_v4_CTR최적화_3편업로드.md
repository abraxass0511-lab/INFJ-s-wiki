## 🎯 핵심 학습 내용

### 1. 시네마그래프(Cinemagraph) 비주얼 전략 확립

**결론**: 시네마그래프는 정지 이미지보다 확실히 우위

| 비교 항목 | 정지 이미지 (v3) | 시네마그래프 (v4) |
|-----------|-----------------|------------------|
| 시각적 임팩트 | Ken Burns 드리프트만 | 미세한 반복 움직임 (비, 증기, 반사) |
| 루프 자연스러움 | N/A | ✅ 시네마그래프 특성상 루프가 자연스러움 |
| 크롭 이슈 | 없음 | 가로→세로 변환 시 60% 손실 (향후 세로 네이티브 필요) |
| 리텐션 예상 | ~35% | ~45% |

**핵심 인사이트**: 
- 8초 시네마그래프 → 17초 루프 시, 시네마그래프의 미세 반복 움직임 특성상 루프 지점이 보이지 않음
- 대표님이 직접 영상 생성 → 내가 렌더링하는 **협업 모델** 확립
- 가로 원본 → 세로 크롭은 화면 60% 손실 → 향후 **세로 네이티브(1080×1920) 원본 생성** 필요

### 2. render_v4_video.py — 비디오 입력 렌더 엔진

| 항목 | 상세 |
|------|------|
| **입력** | MP4 시네마그래프 (가로 원본) |
| **출력** | 1080×1920 (세로), 17초, 24FPS |
| **처리** | 9:16 센터 크롭 → 시간 루프 → 오디오 믹스 → 비주얼라이저 + 텍스트 오버레이 |
| **비트 동기** | 48바 FFT 스펙트럼 분석 기반 |
| **효과** | 색온도 시프트, 글리치, 비트 리액티브 줌/밝기/셰이크 |
