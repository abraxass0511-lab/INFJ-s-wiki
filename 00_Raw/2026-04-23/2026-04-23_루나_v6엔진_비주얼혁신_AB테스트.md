# [[v6 엔진 비주얼 혁신 & A/B 테스트 배포]]

## 📌 Brief Summary
최근 영상 3개의 퍼포먼스를 YouTube Data API v3로 분석해 "정지 이미지 1장 슬라이드쇼" 구조적 한계를 확인하고, Render Engine v6(멀티씬 비트싱크 컷)을 구축하여 AI 이미지 vs 실사 영상 2가지 프로토타입을 A/B 테스트로 예약 발행했다.

## 📖 Core Content

### 1. 최근 영상 3개 퍼포먼스 분석 (YouTube Data API v3)

| 영상 | 조회수 | 좋아요 | 댓글 | 문제점 |
|------|--------|--------|------|--------|
| Coffee & Late Night Code ☕ | ~94 | 0 | 1 (자동) | 리텐션 낮음, 비주얼 반복 |
| Headphones On, World Off 🎧 | ~87 | 0 | 1 (자동) | 동일 이미지 구조, 차별화 실패 |
| Watch Ticking at 2 AM ⌚ | ~112 | 1 | 1 (자동) | 제목만 바꾸고 영상은 동일 |

- **핵심 발견:** 댓글 1개는 전부 API로 자동 생성한 Pin 댓글 → 진짜 시청자 참여 **제로**
- **평균 조회수 ~94뷰:** 60개 영상에 구독자 4명 = 알고리즘 추천 자체가 안 되고 있음
- **리텐션 패턴:** 제목에 물리적 오브젝트(Coffee/Headphones/Watch) 활용은 CTR에 미미한 효과
- **근본 원인:** 비주얼이 전부 동일 → 시청자가 "같은 영상" 인식 → 리텐션 하락 → 알고리즘 비추천 악순환

### 2. 문제 진단 — 냉정한 자기평가
- v4 엔진 = 이미지 1장 + Ken Burns → **모든 영상이 동일하게 보이는 치명적 문제**
- Pin 댓글 = API 자동 생성 → "시청자 참여"라고 착각한 오류 인정
- Chill 장르 전환, 추상적 제목 등 메타데이터 최적화는 비주얼 문제를 해결 못함
- **결론: 메타데이터 튜닝이 아니라 콘텐츠 비주얼 자체를 바꿔야 한다**

### 3. Render Engine v6 — 핵심 기술 변경
- 1장 → **5장 씬 로테이션** (4비트마다 전환)
- 트랜지션: Flash / Glitch / Zoom-Punch (비트 동기화)
- 가사 스타일 텍스트 오버레이 (비트마다 1줄씩)
- 네온 그라데이션 비주얼라이저 (시안→마젠타)
- 루프 구조 (마지막→첫 씬 블렌드)
- 테마: Cyberpunk City (프로토타입), 추후 3종 로테이션 예정
- PIL 호환성: `Image.ANTIALIAS` → `Image.Resampling.LANCZOS` 패치 필수

### 4. Phase 1: Pexels 무료 실사 영상 ✅ 성공
- Pexels CDN에서 Python requests로 직접 다운로드
- HD(1080p) 접근 시 403 → SD(360p) fallback 전략으로 해결
- 5개 클립 확보: rain city(0.6MB), coffee(0.8MB), subway(1.3MB), keyboard(3.7MB), skyline(2.6MB)
- MoviePy `VideoFileClip` → 자동 크롭/리사이즈 (1080x1920 세로)
- **결론: 정지 이미지 → 실사 영상 전환은 체감 퀄리티 차이가 크다**

### 5. Phase 2: Wan2.1 로컬 I2V — 절반 성공
- Intel Arc 140V (16GB VRAM) + XPU → `torch.xpu.is_available()` 정상 확인
- **핵심 발견:** I2V(이미지→영상)는 14B 모델만 존재 (~28GB), 1.3B는 T2V(텍스트→영상)만 지원
- `hf_xet` 패키지 미설치 시 HuggingFace HTTP fallback으로 다운로드 극도 저하
- venv_xpu에 pip 미설치 → `ensurepip.bootstrap()` 실행 후 `pip install hf_xet`으로 해결
- T2V 1.3B (~3GB) 모델 다운로드 진행 중 (미완료, 다음 세션에서 연속)
- 14B I2V 모델은 이전 시도에서 1.67GB/28GB까지 캐시됨 (재개 가능)

### 6. A/B 테스트 배포
- 영상 1 (AI 이미지 5장 비트싱크): ID `t2-PUWlQVG0` → 2026-04-24 02:00 KST (US EST 1PM)
- 영상 2 (실사 스톡 5개 비트싱크): ID `oj2D733AL2o` → 2026-04-24 08:00 KST (US EST 7PM)
- **비교 축:** AI 이미지 멀티씬 vs 실사 영상 멀티씬 vs 기존 v4 정지이미지 1장
- 48시간 후 리텐션/조회수 비교하여 방향 결정
- CTA 전략: 영상1=구독 유도, 영상2=댓글 유도("Which scene hits different?")

### 7. 냉정한 자기 평가 (가장 중요)
- v6는 v4 대비 **30~50% 개선 예상**이지, 게임 체인저는 아님
- 정지 이미지 5장도 결국 "사진" — 시청자는 0.5초 만에 판단
- PIL 텍스트 렌더링(Arial+stroke)은 CapCut 모션 타이포 대비 아마추어 수준
- 채널의 근본 문제는 비주얼이 아니라 **"이 채널만의 가치"** 부재
- AI 자동화를 맹신하지 말고, 시청자 데이터(리텐션)가 정답
- 예측: v6 AI이미지 ~120-150뷰, v6 실사 ~150-200뷰 (기존 v4 ~94뷰)
