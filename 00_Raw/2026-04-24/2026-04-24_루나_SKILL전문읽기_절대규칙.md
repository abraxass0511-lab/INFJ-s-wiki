# [[SKILL.md 전문 읽기 절대 규칙]]

## 📌 Brief Summary
에이전트가 SKILL.md를 grep 부분 검색만 하고 전문을 읽지 않아, 모든 규칙(황금 시간대, 분 jitter, 제목 규칙, 장르 매핑)을 빼먹은 사건. 발행 시간을 3번이나 수정하는 사태 발생. 대표님 직접 지시로 "SKILL.md 전문 읽기"를 최상위 절대 규칙으로 등록.

## 📖 Core Content

### 사건 (2026-04-24)
1. 영상 3개 제작 완료 후 발행 스케줄 설정 시작
2. 에이전트가 SKILL.md에 명시된 슬롯 스케줄(KST 21:00/01:00/06:00)을 무시하고 임의 시간 제안
3. 대표님이 "skill에 황금시간대 있잖아" 지적 → grep 검색으로 찾아서 수정
4. 분 jitter(봇 탐지 방지)도 빼먹음 → 대표님 재지적
5. 3슬롯 배치 대신 3일×Slot2 로 잘못 업로드 → 예약 시간 재수정
6. 총 3번 수정, 대표님 시간 10분+ 낭비

### 근본 원인
- SKILL.md를 세션 시작 시 전문 읽기하지 않음
- 필요할 때 grep으로 부분 검색만 함
- 검색 결과만 보고 전체 맥락(다른 규칙과의 연관)을 놓침

### 절대 규칙 (SKILL.md Section 0 최상단에 등록)
1. 세션 시작 시 SKILL.md를 view_file로 1행~마지막행 전문 읽기 (800줄 이하이므로 1회 호출)
2. 작업 전 체크리스트 자기검증 (슬롯/jitter/제목/장르/음악이미지독립/WAV변환/Python분리)
3. grep 부분 검색만으로 규칙 확인 완료로 판단하는 것 금지
4. KNOWLEDGE_BASE.md, prompt_templates.json도 함께 읽기

### 교훈
- "읽었다"와 "적용했다"는 다르다
- grep 검색은 **보조 수단**일 뿐, 전문 읽기를 대체할 수 없다
- 규칙은 독립적이 아니라 서로 연결되어 있으므로 전체를 알아야 개별 규칙을 올바르게 적용할 수 있다
