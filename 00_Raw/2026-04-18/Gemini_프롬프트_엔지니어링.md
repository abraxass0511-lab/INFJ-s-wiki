# Gemini API 프롬프트 엔지니어링 워크플로우

## 목적
Gemini 3.1 Pro 모델을 활용한 효과적인 프롬프트 작성법과 반복 개선 워크플로우.

## 기본 템플릿
```
# Role: [역할 정의]
# Task: [구체적 작업 설명]
# Constraints: [제약 조건]
# Output Format: [출력 형식 지정]
# Examples: [Few-shot 예시]
```

## 워크플로우 단계
1. **초안 작성**: 역할 + 작업 + 제약 조건으로 초안 작성
2. **테스트**: 3-5개 다양한 입력으로 테스트
3. **개선**: 실패 케이스 분석 → 제약 조건 추가
4. **자동화**: 검증된 프롬프트를 코드에 하드코딩

## 핵심 기법
- **Chain-of-Thought**: "단계별로 생각해줘" 보다 구체적 단계를 명시
- **Few-shot**: 최소 2-3개 예시 제공 (품질 > 수량)
- **JSON 출력 강제**: `Output must be valid JSON. No additional text.`
- **온도 제어**: 창의적 작업은 0.9, 구조화 작업은 0.1-0.3

## 실전 팁
> 프롬프트는 "코드"처럼 버전 관리하라. 작동하는 프롬프트를 변경할 때는 반드시 이전 버전을 백업하라.
