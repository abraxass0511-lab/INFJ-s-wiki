# [[사진/동영상 전송기 — FastAPI 서버 + 외장하드 자동 이관 개발 전체 기록]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> FastAPI로 사진/동영상 업로드 서버를 만들면 핸드폰 Wi-Fi 전송 → 노트북 저장 → 외장하드 자동 이관이 가능하지만, 웹 브라우저는 핸드폰 갤러리 접근/삭제/썸네일 생성이 불가하므로 이를 위해서는 Flutter 네이티브 앱(APK) 개발이 필요하다. 안드로이드 브라우저는 파일 선택 시 원본 파일명 대신 숫자 ID를 보내므로 서버에서 EXIF 날짜 기반으로 파일명을 복원해야 한다.

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴
- **FastAPI 멀티파트 업로드** — `UploadFile`로 500MB까지 사진/동영상 수신, 사진과 동영상 분기 처리 (EXIF/썸네일/얼굴인식은 사진만)
- **SHA-256 해시 중복방지** — 파일 해시로 DB에서 중복 확인, 단 실제 파일이 삭제되었으면 DB 정리 후 재전송 허용
- **외장하드 자동 이관** — psutil로 5초마다 드라이브 마운트 감시, 연결 감지 시 버퍼→외장하드 자동 복사
- **폴더 감시** — 웹 업로드뿐 아니라 폴더에 직접 넣은 파일도 자동 이관 (MEDIA_EXTENSIONS 필터)
- **안드로이드 파일명 문제** — 브라우저 파일 피커가 content provider ID(숫자)를 보냄 → EXIF 날짜 기반 `IMG_YYYYMMDD_HHMMSS` 형식으로 복원
- **Pydantic Optional 필드** — 동영상은 썸네일이 없으므로 `thumbnail_path: str | None = None`으로 선언 필수
- **PowerShell 한글 경로** — cmd.exe batch 파일은 한글 경로 인코딩 깨짐 → .ps1 스크립트 + 바탕화면 바로가기(.lnk)로 우회
- **웹 vs 네이티브 앱 한계** — 웹 브라우저는 보안상 갤러리 접근/파일 삭제/썸네일 생성 불가 → Flutter 앱 필요

### 대화 5: 파일 삭제 후 재전송 불가

**문제:** 폴더에서 파일을 지워도 DB 기록이 남아 "이미 전송됨" 거부

**수정 — 스마트 중복 체크:**
```python
# server/routers/upload.py
existing = await db.fetch_one("SELECT id FROM photos WHERE id = ?", (file_hash,))
if existing:
    buf_path = Path(settings.TEMP_BUFFER_PATH) / file.filename
    if buf_path.exists():
        raise HTTPException(409, "이미 전송된 파일입니다")
    else:
        # 파일 삭제됨 → DB 정리 후 재전송 허용
        await db.execute("DELETE FROM buffer_queue WHERE photo_id = ?", (file_hash,))
        await db.execute("DELETE FROM photos WHERE id = ?", (file_hash,))
```

### 대화 7: 대기 숫자가 실제 파일과 불일치

**문제:** 폴더가 비어있는데 대기 3개로 표시

**수정 — 실제 파일 존재 확인:**
```python
# server/services/buffer_service.py
async def get_pending_count(self):
    pending = await db.fetch_all("SELECT ... FROM buffer_queue WHERE status IN ('pending','failed')")
    actual_count = 0
    for item in pending:
        if Path(item["buffer_file_path"]).exists():
            actual_count += 1
        else:
            # 파일 삭제됨 → DB 자동 정리
            await db.execute("DELETE FROM buffer_queue WHERE id = ?", (item["id"],))
            await db.execute("DELETE FROM photos WHERE id = ?", (item["photo_id"],))
    return actual_count
```

### 대화 8: 폴더 직접 파일 넣기 자동 이관

```python
# server/utils/drive_watcher.py
MEDIA_EXTENSIONS = {".jpg", ".jpeg", ".png", ".mp4", ".mov", ...}

async def _transfer_folder_files(self):
    media_files = [f for f in self.buffer_path.iterdir()
                   if f.is_file() and f.suffix.lower() in MEDIA_EXTENSIONS]
    for file_path in media_files:
        result = storage_service.save_to_external(str(file_path), target_folder, file_path.name)
        if result:
            file_path.unlink()  # 원본 삭제
```

### 대화 9: 서버 실행 방법 — 바탕화면 바로가기

**문제:** cmd.exe batch 파일이 한글 경로를 인코딩 못함

**시도 및 결과:**
| # | 시도 | 결과 |
|---|------|------|
| 1 | `.bat` 파일 (`chcp 65001`) | ❌ 한글 깨짐 |
| 2 | `.bat` → PowerShell 호출 | ❌ 경로 파싱 실패 |
| 3 | 8.3 짧은 경로명 사용 | ❌ 한글 short path도 깨짐 |
| **4** | **`.ps1` 스크립트 + `.lnk` 바로가기** | **✅ 성공** |

**최종 구조:**
```
start_server.ps1          ← PowerShell 스크립트 (프로젝트 내)
사진전송서버.lnk          ← 바탕화면 바로가기 (ps1 실행)
```

**start_server.ps1 핵심:**
```powershell
$ip = (Get-NetIPAddress -AddressFamily IPv4 |
       Where-Object { $_.IPAddress -notmatch "^(127\.|169\.254)" } |
       Select-Object -First 1).IPAddress
Write-Host "  Phone Access URL:" -ForegroundColor Green
Write-Host "  http://${ip}:8000/web/" -ForegroundColor White
Set-Location "$PSScriptRoot\server"
& .\venv\Scripts\python.exe main.py
```

**IP 감지 주의점:**
- `169.254.x.x` (APIPA) 제외 필수 → 실제 Wi-Fi IP만 표시
- 핸드폰과 노트북이 같은 Wi-Fi에 연결되어야 접속 가능

### 대화 10: 안드로이드 파일명 숫자 ID 문제

**문제:** 핸드폰에서 `Screenshot_20260528_184917_Google.jpg` 전송 → 서버에 `1000035580.jpg`로 도착

**원인:** 안드로이드 브라우저의 content provider가 원본 파일명 대신 media store ID를 전달

**수정 — EXIF 날짜 기반 파일명 복원:**
```python
# server/routers/upload.py
import re
from datetime import datetime

stem = Path(file.filename).stem
suffix = Path(file.filename).suffix

if re.match(r'^\d+$', stem):  # 순수 숫자 = 안드로이드 ID
    if exif_data["taken_at"]:
        dt = datetime.fromisoformat(exif_data["taken_at"])
        original_name = f"IMG_{dt.strftime('%Y%m%d_%H%M%S')}{suffix}"
    else:
        original_name = f"IMG_{file_hash[:8]}{suffix}"
```

**결과:**
| 핸드폰 전송 | 저장 파일명 |
|------|------|
| `1000035580.jpg` | `IMG_20260528_184917.jpg` |
| `Screenshot_2026...jpg` | 그대로 유지 |

### 대화 15: 웹 사이트와 앱의 관계

**사용자:** "앱을 만들거면 사이트는 쓸모없는거야?"

**답변:** 둘 다 유용

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **"웹으로 모든 것이 가능하다" 오해:** 웹 브라우저는 보안 정책으로 갤러리 접근/파일 삭제 불가 → 네이티브 앱 필수
- **"원본 파일명이 유지된다" 오해:** 안드로이드 브라우저는 content provider ID를 보냄 → 서버에서 EXIF 기반 복원 필요
- **"batch 파일이면 충분하다" 오해:** cmd.exe는 한글 경로 인코딩 깨짐 → PowerShell + 바로가기 필요
- **"DB 해시 중복 체크면 충분하다" 오해:** 파일이 삭제되어도 DB 기록이 남아 재전송 차단 → 실제 파일 존재 확인 필수
- **"외장하드 감시만으로 충분하다" 오해:** 웹 업로드 파일만 DB에 등록됨 → 폴더 직접 파일도 감시하는 기능 추가 필요
- **"169.254.x.x도 유효한 IP" 오해:** APIPA(자가 할당 IP)로 외부 기기에서 접근 불가 → Wi-Fi IP만 표시
