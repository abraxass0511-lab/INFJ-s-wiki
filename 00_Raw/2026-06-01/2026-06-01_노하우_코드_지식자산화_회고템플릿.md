# [[지식 자산화 — 노하우와 코드의 데이터화 — 챕터 9]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 코드 자체는 "지식"이 아니다. 코드 = 결과물. 지식 = 그걸 만들 때 머릿속에서 내린 의사결정.

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴
- **노하우 5가지 각도** — 직접(What/How), 상황(When/Where), 초보(Why), 반대(But), 잡담(Feel)
- **1노하우 = 10~20 Q&A** — 한 덩어리 지식을 5가지 각도에서 펼쳐야 데이터가 됨
- **코드 지식화 5카테고리** — 의사결정(Why), 삽질(Mistakes), 패턴(How), 함정(Avoid), 메타 교훈(Wisdom)
- **회고 템플릿 6개 질문** — 30분 투자 → 30~50개 Q&A 자동 생성
- **AI 자동 추출 프롬프트** — 코드를 AI에게 보여주고 5가지 카테고리로 20개 Q&A 요청 → 30분 회고가 1분으로 단축

### 세부 내용
- **노하우 → 데이터 전환 테이블:**

| 각도 | 의도 | 예시 질문 |
| --- | --- | --- |
| **직접** (What/How) | 정체성 박기 | "외주 단가 얼마부터?" |
| **상황** (When/Where) | 실전 응용 | "클라가 단가 깎아달래" |
| **초보** (Why) | 친절한 톤 | "처음 어떻게 정해?" |
| **반대** (But) | 신념 강화 | "5만 원은 너무 비싸지 않아?" |
| **잡담** (Feel) | 페르소나 톤 | "단가 정하기 너무 어려워" |

- **코드/프로젝트 지식화 테이블:**

| 카테고리 | 내용 |
| --- | --- |
| **의사결정** (Why) | 왜 이 기술을 골랐는지 |
| **삽질/실수** (Mistakes) | 어디서 막혔는지 |
| **패턴** (How) | 반복 가능한 구조 |
| **함정** (Avoid) | 다른 사람이 실수할 부분 |
| **메타 교훈** (Wisdom) | 코드 너머의 원칙 |

- **회고 템플릿 (프로젝트당 30분):**
```markdown
1) 왜 만들었나? (1줄)
2) 어떤 기술 골랐고 왜?
3) 막혔던 순간 Top 3
4) 다음에 만들면 다르게 할 것 3가지
5) 처음부터 알았으면 좋았을 것 5가지
6) 다른 프로젝트에도 쓸 패턴 3가지
```

- **AI 자동 추출 프롬프트:**
```
이 코드는 내가 짠 것이다. 이 코드를 짤 때 내가 내린 의사결정과
삽질했을 만한 부분을 분석해서, 5가지 카테고리로 골고루 20개 Q&A 생성:

1. 의사결정 (Why)
2. 삽질 (Mistakes)
3. 패턴 (How)
4. 함정 (Avoid)
5. 메타 교훈 (Wisdom)

답변 톤:
- 친근하고 명확한 1인칭
- "당장 해볼 것:" 같은 액션 포함

[코드 붙여넣기]
```

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **"코드를 복붙하면 문서화다" 오해:** 코드 자체는 결과물일 뿐. 왜 그 기술을 골랐는지(의사결정)가 진짜 지식 → 5가지 카테고리로 추출해야 함
- **"노하우 하나면 Q&A 하나" 오해:** 하나의 노하우를 5가지 각도로 펼쳐야 모델이 다양한 질문에 반응할 수 있음 → 최소 10~20개 Q&A
