## 📌 한 줄 통찰 (The Karpathy Summary)

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴
- **파인튜닝 = 사투리 가르치기** — 처음부터 학습(10년) vs 기존 모델에 미세 조정(1시간)
- **Unsloth 올인원** — Triton 커널(2× 속도) + 4bit 양자화(70% 메모리 절감) + chat template + GGUF 변환 내장
- **LoRA 0.6% 학습** — W_new = W₀(동결❄️) + B·A(학습🔥), rank=16이면 페르소나 충분
- **30개 Q&A sweet spot** — 카테고리 균형이 핵심: 정체성 20%, 전문분야 40%, 태도 20%, 사례 15%, 잡담 5%
- **핵심 파라미터 3개만** — learning_rate=3e-4, max_steps=60, lora_alpha=32
- **Response-only Masking** — 질문은 입력만, 답변에만 loss 계산 → 효율↑ 품질↑
- **bos 토큰 일관성** — 학습 형식 = 추론 형식, 1 토큰 차이로 LoRA 효과 0%
- **하이브리드 아키텍처** — Gemini(도구/추론) + 로컬 Gemma(페르소나) = AI 1인 기업 표준 패턴

from unsloth import FastModel
model, tokenizer = FastModel.from_pretrained(
    model_name = "unsloth/gemma-4-E2B-it",
    max_seq_length = 1024,
    load_in_4bit = True,
    full_finetuning = False,
)
```

- **데이터 형식 (HuggingFace 표준):**

{"conversations": [

]}
```

- **Loss 해석 가이드:**

| Loss 값 | 의미 | 조치 |
| --- | --- | --- |
| > 1.0 | 학습 부족 | step ↑ |
| **0.2 ~ 0.5** | ✅ **이상적** | 그대로 사용 |
| < 0.01 | ⚠️ 과적합 | step ↓ |

- **디버깅 3대 함정:**
  - **함정 1 — bos 토큰 불일치:** 학습 시 `<|turn>user...` (bos 없음) vs 추론 시 `<bos><|turn>user...` (bos 자동 추가) → 1 토큰 차이로 LoRA 효과 100%→0%
  - **함정 2 — 과적합과 답변 반복:** Loss < 0.01이면 분포 붕괴, 입력 echo 현상 → max_steps=60 + 정체성 변형 데이터 5개 추가
  - **함정 3 — GGUF 변환 RAM 부족:** 시스템 RAM(12.7GB) < GGUF 필요량(15GB) → Colab Pro 또는 HF Space 변환

- **bos 토큰 제거 해결 코드:**
```python
if inp["input_ids"][0, 0].item() == tokenizer.bos_token_id:
    inp["input_ids"] = inp["input_ids"][:, 1:]
    inp["attention_mask"] = inp["attention_mask"][:, 1:]
```

| # | 체크 항목 | 확인 방법 |
| --- | --- | --- |
| 1 | 토큰 형식 일치 | `print(dataset[0]["text"])` |
| 2 | 마스킹 정상 | 답변만 보여야 함 |
| 3 | LoRA 학습됨 | LoRA 가중치 norm 확인 |
| 4 | 추론 모드 전환 | `FastModel.for_inference(model)` |
| 5 | bos 일관성 | 학습 형식 == 추론 형식 |

- **7가지 핵심 교훈:**
  1. LoRA는 0.6% 학습으로 페르소나를 박는다
  2. 신경망은 "표현"을 외우지 "의미"를 외우지 않는다
  3. 하이퍼파라미터의 핵심은 3개 — lora_alpha, learning_rate, max_steps
  4. Loss 0.0001은 좋은 게 아니다 — 과적합 = 분포 붕괴
  5. 학습 형식 = 추론 형식 (1 토큰도) — bos 한 토큰 차이로 전부 무너진다
  6. 메모리는 GPU만이 아니다 — 시스템 RAM이 GGUF 변환의 진짜 병목
  7. 모델은 소비재, 데이터는 자산

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **"0.6%만 학습하면 결과가 안 좋다" 오해:** LoRA 논문(2021)이 증명 — 파인튜닝에서 의미 있는 가중치 변화는 매우 저랭크(low-rank), 풀 파인튜닝과 거의 같은 결과
- **"Loss가 낮을수록 좋다" 오해:** Loss < 0.01이면 과적합 → 분포 붕괴, 학습 표현만 인식, 변형 표현 무시
- **"GPU 메모리만 신경 쓰면 된다" 오해:** 시스템 RAM이 GGUF 변환의 진짜 병목 → T4 무료(12.7GB) < 필요량(15GB)
- **"Antigravity SDK에서 로컬 모델 직접 호출 가능" 오해:** Gemini 전용 → 하이브리드 아키텍처로 해결 (라우팅)
