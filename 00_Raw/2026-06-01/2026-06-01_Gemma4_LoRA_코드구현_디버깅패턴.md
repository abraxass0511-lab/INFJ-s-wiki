## 📌 한 줄 통찰 (The Karpathy Summary)
> Gemma 4 LoRA 파인튜닝의 핵심은 코드 자체가 아니라, bos 토큰 1개 차이로 LoRA 효과가 0%가 되고, Loss 0.0001은 좋은 게 아니라 과적합이며, 시스템 RAM이 GGUF 변환의 진짜 병목이라는 의사결정과 삽질의 기록이다.

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴
- **모델 로딩 패턴** — `FastModel.from_pretrained(load_in_4bit=True, full_finetuning=False)` → T4(15GB)에서 학습 가능
- **LoRA 부착 패턴** — `get_peft_model(r=16, lora_alpha=32)` → 전체의 0.6%만 학습
- **데이터 형식 패턴** — HuggingFace 표준 `{"conversations": [{"role":"user","content":...}]}`
- **채팅 템플릿 패턴** — `get_chat_template(tokenizer, "gemma-4")` + `.removeprefix('<bos>')` 중복 방지
- **Response-only Masking** — `train_on_responses_only(instruction_part="<|turn>user\n", response_part="<|turn>model\n")`
- **bos 토큰 제거 패턴** — 추론 시 `if inp["input_ids"][0,0] == tokenizer.bos_token_id` → 첫 토큰 제거
- **베이스라인 비교 패턴** — Before(학습 전) → After(학습 후) → 새 질문(일반화) 3단계 검증
- **GGUF 변환 패턴** — `push_to_hub_gguf(quantization_method="q4_k_m")`

- **chat() 추론 함수 (bos 제거 포함):**
```python
def chat(prompt, max_tokens=200):
    msg = [{"role": "user", "content": [{"type": "text", "text": prompt}]}]
    inp = tokenizer.apply_chat_template(
        msg, add_generation_prompt=True, tokenize=True,
        return_dict=True, return_tensors="pt"
    ).to("cuda")

    # ⭐ bos 토큰 제거 — 학습 형식과 일치시키기
    if inp["input_ids"][0, 0].item() == tokenizer.bos_token_id:
        inp["input_ids"] = inp["input_ids"][:, 1:]
        inp["attention_mask"] = inp["attention_mask"][:, 1:]

    out = model.generate(**inp, max_new_tokens=max_tokens,
                         do_sample=False, pad_token_id=tokenizer.eos_token_id)
    answer = tokenizer.decode(out[0][inp["input_ids"].shape[1]:], skip_special_tokens=True)
    print(f"❓ {prompt}\n💬 {answer}")
```

  2. **어떤 기술 골랐고 왜?** Unsloth(무료 T4 최적화), LoRA(0.6% 학습), SFTTrainer(HF 표준), GGUF q4_k_m(최적 양자화)
  3. **막혔던 순간 Top 3:** bos 토큰 불일치, 과적합(Loss 0.0001), GGUF RAM 부족

  6. **다른 프로젝트에도 쓸 패턴 3가지:** bos 제거 패턴, Response-only Masking, Before/After 비교 검증

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **"FileResponse면 동영상도 된다" 식의 과신 주의:** 학습에서도 마찬가지로 apply_chat_template이 자동으로 bos를 추가하는 걸 간과하면 전체 실패
- **"Loss가 낮으면 좋다" 오해:** 0.0001은 과적합 → 분포 붕괴 → 적정값 0.2~0.4
- **"max_steps를 늘리면 더 잘 학습된다" 오해:** 30개 데이터에 120+ steps는 과적합 → 60 steps(8 epoch)이 최적
