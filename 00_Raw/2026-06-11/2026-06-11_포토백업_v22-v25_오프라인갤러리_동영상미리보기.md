# [[포토백업 v22~v25 — 오프라인 갤러리 & 동영상 미리보기 구현]]

## 📌 한 줄 통찰 (The Karpathy Summary)
> 서버 의존을 끊는 핵심은 "로컬 파일이 있으면 무조건 로컬 우선"이다. 서버 상태 변수에 의존하면 반드시 엣지케이스에서 실패한다.

## 📖 구조화된 지식 (Synthesized Content)

### 추출된 패턴

- **오프라인 우선 원칙** — 서버 상태(`isServerOnline`)에 의존하지 말고, 로컬 파일 존재 여부로 판단해야 함
- **썸네일 로컬 캐싱** — 동기화 시 서버 썸네일을 핸드폰 앱 내부 저장소에 복사 (약 15MB/1,300장)
- **동영상 미리보기 생성** — FFmpeg로 원본 동영상을 360p MP4로 변환하여 핸드폰에 저장
- **로컬 경로 보존** — 동기화 시 `clearPhotos()` → `bulkUpsertPhotos()` 과정에서 로컬 경로가 날아가는 문제 → 삭제 전 백업 → 재삽입 시 복원
- **병렬 다운로드** — 썸네일 1,298장을 순차(느림) → 10개씩 병렬(10배 빠름)로 다운로드
- **오프라인 삭제** — 로컬 DB 즉시 삭제 → 서버는 오프라인 큐에 저장 → 다음 동기화 시 전송
- **sqlite3.Row 주의** — Python `sqlite3.Row` 객체는 딕셔너리의 `.get()` 메서드를 지원하지 않음 → `AttributeError` 발생

### 세부 내용

- **아키텍처 (데이터 흐름):**

```
새 동영상 전송 → 서버에서 360p 미리보기 자동 생성 (FFmpeg)
      ↓
앱 동기화 → 썸네일 + 미리보기를 핸드폰 앱 내부 저장소에 다운로드
      ↓
서버 꺼짐 → 로컬 썸네일로 갤러리 즉시 표시 + 로컬 미리보기로 동영상 재생
```

```dart
if (!syncProvider.isServerOnline) {
  return _buildPlaceholder(photo); // 서버 상태가 캐시되어 엣지케이스 발생

// ✅ 올바른 방식 (로컬 파일 존재 우선)
if (photo.localThumbnailPath != null) {

  if (file.existsSync()) {
    return Image.file(file, fit: BoxFit.cover); // 무조건 로컬
  }
}
// 로컬 없을 때만 서버 시도
return CachedNetworkImage(imageUrl: thumbUrl, ...);
```

## ⚠️ 모순 및 업데이트 (Contradictions & RL Update)
- **"`isServerOnline`으로 오프라인 처리" 오해:** 서버 상태 변수는 마지막 체크 시점의 캐시일 뿐 → 실시간 상태를 반영 못 함 → 로컬 파일 존재 여부가 유일하게 신뢰할 수 있는 판단 기준
- **"동기화 = 전체 교체" 부작용:** `clearPhotos()` → 재삽입 시 앱 전용 필드(`localThumbnailPath`)가 사라짐 → 동기화 전 앱 전용 필드를 별도 백업해야 함
- **"순차 다운로드면 충분하다" 오해:** 1,298장 순차 다운로드는 수 분 소요 → 10개씩 병렬 처리로 체감 10배 향상
